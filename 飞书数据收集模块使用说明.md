# 飞书妙搭 + 多维表格 数据收集模块使用说明

## 📦 快速开始

### 步骤1：在游戏HTML中引入脚本

在每个游戏的 `index.html` 的 `<head>` 或 `<body>` 末尾添加：

```html
<script src="../feishu-game-data-collector.js"></script>
```

**注意**：根据游戏目录层级调整路径，例如：
- 游戏0：`<script src="../feishu-game-data-collector.js"></script>`
- 主页：`<script src="feishu-game-data-collector.js"></script>`

### 步骤2：游戏关卡切换时更新数据

在关卡切换或完成时，调用全局API更新关卡数据：

```javascript
// 方式1：完整设置
FeishuGameData.setLevel(
    2,           // levelId: 关卡编号
    '平面与投影', // levelName: 关卡名称
    92,          // score: 得分
    true         // completed: 是否完成
);

// 方式2：只设置当前关卡
window.currentLevelId = 2;
window.currentLevelName = '平面与投影';
window.currentScore = 92;
window.levelCompleted = true;
```

### 步骤3：学生点击提交

右下角会自动出现一个浮动的"📤 提交成绩到飞书"按钮，学生点击即可提交。

## 🎯 核心功能

### 1. 自动识别游戏信息

脚本会自动从URL路径中识别：
- `gameId`：游戏编号（如"游戏0"）
- `gameName`：游戏名称（如"画法几何与建筑制图基础"）

已内置所有19个游戏的名称映射。

### 2. 接收学生身份信息

自动监听来自妙搭的 `FEISHU_STUDENT_INFO` 消息：
- 收到消息后保存到 `localStorage`
- 按钮状态从"⚠️ 请先填写信息" → "📤 提交成绩到飞书"

### 3. 自动收集数据

每次提交时自动收集以下数据：

| 字段 | 来源 | 说明 |
|------|------|------|
| `studentId` | localStorage | 学生学号 |
| `studentName` | localStorage | 学生姓名 |
| `className` | localStorage | 所在班级 |
| `gameId` | URL路径 | 游戏编号 |
| `gameName` | 内置映射 | 游戏全称 |
| `levelId` | 全局变量/DOM | 当前关卡 |
| `levelName` | 全局变量/DOM | 关卡名称 |
| `isCompleted` | 全局变量 | 是否完成 |
| `score` | 全局变量/DOM | 得分 |
| `timeSpent` | 计算 | 用时（秒） |
| `operationData` | 全局变量 | 操作记录JSON |
| `submitTime` | 系统时间 | 提交时间 |

### 4. 自动创建提交按钮

如果页面中没有 `id="feishu-submit-btn"` 的按钮，脚本会自动创建一个浮动按钮：
- 位置：右下角固定
- 样式：绿色圆角按钮
- 状态：根据是否有学生信息自动切换

## 🔧 高级用法


## 🔄 与妙搭通信流程

### 第一步：妙搭 → 游戏（发送学生信息）

```javascript
// 在妙搭端执行
iframe.contentWindow.postMessage({
    type: 'FEISHU_STUDENT_INFO',
    studentId: '2024001',
    studentName: '张三',
    className: '建筑学1班'
}, '*');
```

游戏端收到后：
1. 保存到 `localStorage`
2. 启用提交按钮

### 第二步：游戏 → 妙搭（提交成绩）

```javascript
// 自动执行，无需手动写
window.parent.postMessage({
    type: 'FEISHU_GAME_SUBMIT',
    data: { /* 收集的所有数据 */ }
}, '*');
```

### 第三步：妙搭 → 多维表格（写入数据）

妙搭端监听到消息后，调用飞书API写入3个表：
1. 学生信息表（有则更新，无则新增）
2. 游戏进度总览（有则更新，无则新增）
3. 关卡完成记录（每次都新增）
